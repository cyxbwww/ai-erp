# AI 智能销售 ERP 项目 - 可执行检查清单（Executable Checklist）

当新增一个 ERP 模块时，必须逐条检查以下项目是否完成。

---

## 一、数据库
- [ ] 已创建数据表
- [ ] 已设计字段
- [ ] 已设置主键
- [ ] 已设置必要索引
- [ ] 已设置表关系（如客户、订单、商品关联）

---

## 二、后端（FastAPI）
- [ ] 已创建 SQLAlchemy Model
- [ ] 已创建 Pydantic Schema
- [ ] 已创建 CRUD 接口
- [ ] 列表接口支持分页
- [ ] 列表接口支持关键词搜索
- [ ] 列表接口支持状态筛选（如有状态字段）
- [ ] 接口返回格式为 { code, message, data }
- [ ] 已添加基础错误处理
- [ ] 接口路径以 /api 开头
- [ ] 接口命名符合 RESTful 风格

接口示例：
- GET /api/{module}/list
- GET /api/{module}/detail/{id}
- POST /api/{module}/create
- PUT /api/{module}/update/{id}
- DELETE /api/{module}/delete/{id}

---

## 三、前端（Vue3 + Naive UI）
- [ ] 已创建 API 调用文件
- [ ] 已创建列表页
- [ ] 已创建新增表单
- [ ] 已创建编辑表单
- [ ] 已实现删除功能
- [ ] 已创建详情页（如模块需要）
- [ ] 页面包含搜索区域
- [ ] 页面包含表格列表
- [ ] 页面包含分页
- [ ] 表单包含校验
- [ ] 页面包含加载状态
- [ ] 页面包含错误提示
- [ ] 页面 UI 风格统一使用 Naive UI

---

## 四、系统集成
- [ ] 已添加前端路由
- [ ] 已添加侧边栏菜单
- [ ] 已添加权限控制（如需要）
- [ ] 已与客户/商品/订单等模块建立关联（如需要）
- [ ] 已在 Dashboard 中考虑统计数据（如需要）

---

## 五、代码注释检查（强制）
- [ ] 每个新增文件都有文件头注释
- [ ] 每个 Model 有注释说明表作用
- [ ] 每个字段有注释说明含义
- [ ] 每个 API 接口有注释说明用途
- [ ] 每个 Service 有注释说明业务逻辑
- [ ] 每个前端页面有注释说明页面作用
- [ ] 每个 API 文件有注释说明接口用途
- [ ] 核心状态变量有注释说明用途
- [ ] 核心方法有注释说明作用
- [ ] 复杂逻辑有注释说明
- [ ] 关键样式块有注释说明
- [ ] 如缺少注释，则该模块视为未完成

---

## 六、功能测试
- [ ] 可以新增数据
- [ ] 可以编辑数据
- [ ] 可以删除数据
- [ ] 可以查看详情
- [ ] 可以搜索数据
- [ ] 可以分页
- [ ] 前后端字段一致
- [ ] 接口可以正常调用
- [ ] 页面没有明显报错

---

## 七、AI 功能检查（如果该模块包含 AI）
- [ ] AI 功能嵌入在业务页面中（不是单独聊天框）
- [ ] 已创建 AI 接口
- [ ] 已封装 Prompt
- [ ] 前端有 AI 按钮入口
- [ ] AI 返回结果可以展示在页面
