# 订单处理流程

## 1. 订单创建
1. 创建订单前，先校验客户信息与商品明细是否完整。
2. 商品信息需包括商品名称、商品编码、单价、数量和单位。
3. 若客户信息不完整，应先补充客户联系人、手机号和负责人信息。
4. 创建订单后，系统进入“草稿”状态。

## 2. 订单确认
1. 草稿订单需要业务人员确认后，方可进入“已确认”状态。
2. 确认订单前应重点检查：
   - 客户信息是否正确
   - 商品数量和金额是否正确
   - 支付方式是否明确
   - 备注与特殊要求是否已记录
3. 已确认订单表示订单已进入履约准备阶段。

## 3. 支付处理
1. 已确认订单需跟踪付款状态。
2. 若客户已付款，应及时登记支付方式、支付时间和交易单号。
3. 若长时间未付款，应进行催付提醒，并记录沟通结果。
4. 对于金额较高的订单，应重点关注付款节点与异常情况。

## 4. 发货处理
1. 已确认且满足发货条件的订单，进入发货处理阶段。
2. 发货时需记录：
   - 物流公司
   - 物流单号
   - 发货时间
3. 若商品库存不足，应先协调补货或调整发货计划。
4. 发货后应同步更新订单状态，避免前后信息不一致。

## 5. 完成订单
1. 商品已发货并完成交付后，可将订单标记为“已完成”。
2. 完成订单前应确认：
   - 支付已完成
   - 发货信息完整
   - 客户无异常反馈
3. 已完成订单可进入统计分析与复盘阶段。

## 6. 取消订单
1. 若客户主动取消、付款异常或业务判断无法继续履约，可取消订单。
2. 取消订单前应明确原因，并在备注中记录。
3. 若订单已占用库存，应同步处理库存回滚。
4. 若订单已进入支付流程，应同步确认财务处理状态。

## 7. 异常订单处理
以下情况可视为异常订单：
- 长时间未付款
- 高金额订单但客户信息不完整
- 商品数量异常
- 发货延迟
- 取消率较高客户重复下单

异常订单建议处理方式：
1. 优先人工复核
2. 联系客户确认真实需求
3. 核对付款与发货状态
4. 记录异常原因并补充处理结果

## 8. 风险处理建议
1. 连续超过 3 天未付款的订单，建议升级提醒。
2. 金额较高订单建议加入人工复核。
3. 对重复取消订单客户建议复盘成交策略。
4. 对高风险订单应在系统中标记并记录处理经过。
